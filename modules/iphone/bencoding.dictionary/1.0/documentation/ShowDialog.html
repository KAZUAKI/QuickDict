benCoding.Dictionary Module

Method: showDialog

Description

The showDialog method opens a modal dialog with the definition of the term provided. If the dictionary service is unable to find the definition the words "No definition found." will be displayed.

Accessing the benCoding.Dictionary Module

Before you can start using any of the module's functionality you need to use the require keyword to make the module accessible from JavaScript. To access this module from JavaScript, you would do the following:

var referenceLibrary = require('bencoding.dictionary').createReferenceLibrary();

Using showDialog

The below example shows how to create the ReferenceLibrary object and check if the term provided, "life", has a definition available then open a Modal dialog with the definition.

var ourTerm = "life";
function termDialogBoxHasBeenClosed(){
    alert('Term Window Closed');
};
function termDialogHadError(e){
    alert("Term Lookup Errored Due To: " + e.error);
};


if((referenceLibrary.isSupported())&&(referenceLibrary.wordHasDefinition(ourTerm ))){

     Ti.API.info("You can add an event to be called if there is an error");
     referenceLibrary.addEventListener('errored', termDialogHadError);

      Ti.API.info("You can add an event to be called when the definition dialog is closed");
      referenceLibrary.addEventListener('closed', termDialogBoxHasBeenClosed);

      Ti.API.info("Does Apple know the meeting of life?");
       referenceLibrary.showDialog({
             term:ourTerm , //This is the term to search for (REQUIRED)
             animated:true, //Indicate if the dialog should be animated on open (OPTIONAL)
             modalTransitionStyle:Ti.UI.iPhone.MODAL_TRANSITION_STYLE_FLIP_HORIZONTAL //(OPTIONAL)
       });

}else{

     alert('This feature is not supported or the word wasn't found');

};

 

Why do I use this method?

There are a few reasons why you would use this method.  The most common is to check if your term has a definition before calling the showDialog.  

What if I don't use isSupported?

If you call the showDialog and your device does not support the dictionary feature the definition modal dialog will not appear and the errored callback function will be fired (if defined).

What if I the dictionary can't find my term?

If you call the showDialog and the dictionary cannot find the provided term, the modal dialog will appear displaying the message "No definition found.".

Arguments

Name Type Summary
term String Term or word for the dictionary service to find and display the definition for (required)
animated boolean Indicate if the dialog should be animated on open.  Use of this parameter is optional. If no parameter is provided no animation will be performed.
modalTransitionStyle Number When the Definition Dialog is dismissed a transition will take place.  You can use any of the MODAL_TRANSITION_STYLE properties available by Titanium. This parameter is optional, if not provided not transition will be performed.

Events

Name Summary
closed Fired when Definition Dialog is closed. No arguments are provided to the callback function.
errored Fired when/if there is an error.  The argument error is provided to the callback function. See the above snippet for an example.
