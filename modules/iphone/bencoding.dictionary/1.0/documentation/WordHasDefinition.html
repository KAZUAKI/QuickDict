benCoding.Dictionary Module

Method: wordHasDefinition

Description

The wordHasDefinition method returns a boolean indicator on whether or not the Apple Dictionary service was able to find the provided definition.

For a detailed example please see the app.js included in the example folder.

Accessing the benCoding.Dictionary Module

Before you can start using any of the module's functionality you need to use the require keyword to make the module accessible from JavaScript. To access this module from JavaScript, you would do the following:

var referenceLibrary = require('bencoding.dictionary').createReferenceLibrary();

Using wordHasDefinition

The below example shows how to create the ReferenceLibrary object and check if the term provided, "life", has a definition available.

if(referenceLibrary.isSupported()){

    Ti.API.info("Does Apple know the meeting of life?");
    var hasDefinition = referenceLibrary.wordHasDefinition('life');
    Ti.API.info("Definition found? => " + hasDefinition);

}else{

     alert('This feature is not supported');

};

 

 

Why do I use this method?

There are a few reasons why you would use this method.  The most common is to check if your term has a definition before calling the showDialog.  

What if I don't use isSupported?

If you call the wordHasDefinition and your device does not support the dictionary feature a boolean false will be returned for any term you lookup.
