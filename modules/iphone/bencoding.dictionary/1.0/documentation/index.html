benCoding.Dictionary Module

Description

Use the native iOS dictionary service within your Titanium application.  This module provides Titanium accessibility to Apple's native UIReferenceLibraryViewController  introduced in iOS5.

Before You Start

  • This module requires iOS 5 or greater
  • This is an iOS module designed to work with Titanium SDK 1.8.1 or greater.
  • Before using this module you first need to install the package. If you need instructions on how to install a 3rd party module please read this installation guide.
  • If you would like access to the source code, please visit us on Github at https://github.com/benbahrenburg/benCoding.Dictionary..

Accessing the benCoding.Dictionary Module

To access this module from JavaScript, you would do the following:

var referenceLibrary = require('bencoding.dictionary').createReferenceLibrary();

Features

Checking If Feature Supported

This method returns a boolean indicator on whether or not Apple's dictionary services are supported. Please note you need to target iOS 5 or greater to use the native UIReferenceLibraryViewController component.

For a detailed description of this method please click here.

The below sample shows how you can check if this feature is supported.

Ti.API.info("This feature is only supported in iOS5 and above");
Ti.API.info("Check if we have the min OS version needed");
var featureSupported = referenceLibrary.isSupported();
Ti.API.info("Is Supported? => " + featureSupported);

Is Definition Available?

This method returns a boolean indicator on whether or not the Apple Dictionary service was able to find the provided definition.

It is recommended that you use this method to first check if there is a definition available before opening a dialog for the definition.

For a detailed description of this method please click here.

The below sample shows how you can check if there is a definition available for your term.

Ti.API.info("Does Apple know the meeting of life?");
var hasDefinition = referenceLibrary.wordHasDefinition('life');
Ti.API.info("Definition found? => " + hasDefinition);

Showing Definition Dialog

The showDialog method opens Apple's Dictionary view for the term provided.

  •  Callback methods are supported for the the "Error" and "On Close of Dialog" events.

  • The Dictionary module also supports the animated and modalTransitionStyle properties.

For a detailed description of this method please click here.

The below same shows how to use the showDialog method. For additional details please see the example and documentation folders within this project.

function termDialogBoxHasBeenClosed(){
    alert('Term Window Closed');
};
function termDialogHadError(e){
    alert("Term Lookup Errored Due To: " + e.error);
};

Ti.API.info("You can add an event to be called if there is an error");
referenceLibrary.addEventListener('errored', termDialogHadError);

Ti.API.info("You can add an event to be called when the definition dialog is closed");
referenceLibrary.addEventListener('closed', termDialogBoxHasBeenClosed);

referenceLibrary.showDialog({
    term:'life', //This is the term to search for (REQUIRED)
    animated:true, //Indicate if the dialog should be animated on open (OPTIONAL)
    modalTransitionStyle:Ti.UI.iPhone.MODAL_TRANSITION_STYLE_FLIP_HORIZONTAL //(OPTIONAL)
});

Methods

Method Name Description
isSupported Returns a boolean whether the dictionary feature is available
wordHasDefinition Returns boolean if the dictionary service is able to find the definition for the provided term.
showDialog Opens Apple's dictionary definition modal window

What Isn't Supported?

The benCoding.dictionary does not support returning a view to attach within a UIPopoverController.

FAQ

Question Answer
Can I "scrape" the definition instead of using the Modal Dialog? Although technically possible this module does not provide this as it is a grey area in Apple's guidance on how to use this feature.
Can I change the style of the dialog? Apple has not yet provided any styling hooks into this dialog.
Is there an Android version? No, this is an iOS only wrapper around the native UIReferenceLibraryViewController component.
